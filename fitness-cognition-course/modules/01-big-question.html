The Big Question

Does how much you exercise today predict how well your brain works tomorrow?

A research project hiding inside Python code

This codebase is an extension of a published scientific paper — Manning et al. (2022) — that asked a simple but profound question:

🧠
The core question: Can we use wearable fitness data — steps, calories, sleep, BMI — to predict how well someone will perform on memory and cognitive tests?

The original paper found correlations using simple statistics. This project goes further: it trains AI models to see whether fitness data from weeks before a brain test can predict performance — suggesting exercise doesn't just correlate with cognition, it may actually cause it.

📊

The Input

Daily fitness readings from wearables: steps, calories burned, active minutes, weight, BMI — tracked for hundreds of days per person.

🧩

The Output

Scores on cognitive tasks: free recall accuracy, temporal clustering, semantic clustering — tests that measure memory and how the brain organizes information.

⏱️

The Twist

Does fitness from 30 days ago predict today's brain test score? 14 days? 90? This lag analysis is the project's key scientific contribution.

The question
Do patterns in the fitness data predict the cognitive scores?
💡
Aha moment: This is what data scientists call a regression problem. Given fitness inputs, predict a cognitive score output. The AI doesn't know the "rules" — it learns them from examples.

The four files that make this work

The entire project lives in four Python files. Think of them like stations on an assembly line — each one does one job, then hands off to the next.

new_code_copy/
📄 reading_data.py ← Load & clean raw CSV files
📄 temporal_dataset.py ← Slice data into time windows
📄 temporal_model.py ← Define & train the AI models
📄 pkl_reading.py ← Load cognitive score files (.pkl)

reading_data.py

The janitor. Raw data from wearables is messy — missing days, zeros that should be blanks, inconsistent formats. This file cleans everything up through a 7-step pipeline.

temporal_dataset.py

The historian. Organizes clean data into overlapping 90-day windows — each one a snapshot of a person's fitness history that the AI will learn from.

temporal_model.py

The brain. Contains two AI architectures (LSTM and Transformer), a training loop, and the lag analysis experiment that is the project's scientific heart.
